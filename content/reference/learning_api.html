Learning API¶

This page documents the stable learning-side entry points that are useful when extending algorithms or runtime configuration.

Use Runtime API for the CLI and orchestration layer. This page focuses on the algorithm and evaluation side.

Overview¶

Module

Role

ctrlp.learning.algorithms

public algorithm package surface and registry helpers

ctrlp.learning.algorithms.config

algorithm normalization, parameter-path resolution, and construction

ctrlp.learning.evaluation

evaluation metrics, summaries, and CSV conversion helpers

ctrlp.learning.algorithms.config.resolve_algo_params_path(algo_name)¶

Resolve the parameters directory path for a given algorithm (dynamic).

Uses dynamic algorithm discovery to determine the parameter path. Heuristics return None (no parameters), while trainable algorithms return their parameter directory path.

Parameters:

algo_name (str) – Algorithm selector (e.g., “bc”, “ppo”, “dqn”)

Returns:

  • Path to the algorithm’s parameters directory, or None for

  • parameter-less algorithms (heuristics)

Return type:

Path | None

Examples

bc -> src/ctrlp/learning/algorithms/imitation/bc/parameters/ ppo -> src/ctrlp/learning/algorithms/reinforcement/model_free/ppo/parameters/ dqn -> src/ctrlp/learning/algorithms/reinforcement/model_free/dqn/parameters/ zigzag -> None (heuristic, no parameters)

ctrlp.learning.algorithms.config.config_algo(names, environment, parameters, device, use_sbx=False)¶

Configure and instantiate an algorithm (dynamic discovery).

Uses the dynamic registry to determine algorithm type and instantiate appropriately. Supports heuristics, RL algorithms (SB3/sb3_contrib/SBX), and custom IL algorithms.

Parameters:

  • names (Dict[str, Any]) – Dict with ‘algo’ key containing algorithm name

  • environment (Any) – Training environment

  • parameters (Dict[str, Any]) – Algorithm hyperparameters

  • device (str | None) – Device to run on (‘auto’, ‘cuda’, ‘cpu’)

  • use_sbx (bool) – Use JAX-based algorithms from SBX

Returns:

Instantiated algorithm ready for training/evaluation

Return type:

Any

ctrlp.learning.evaluation.resolve_metric_config(raw, defaults=None)¶

Resolve metric configuration from various input formats.

Supports: - None: Use default metrics - String: Single metric (e.g., “reward” or “median_reward”) - List: Multiple metrics (e.g., [“median_reward”, “mean_success”]) - Dict: Explicit configuration

The cleaner syntax “operation_metric” (e.g., “median_reward”, “mean_length”) is now preferred and will be parsed automatically.

Parameters:

  • raw (str | Iterable[str] | Dict[str, Any] | None)

  • defaults (Dict[str, str] | None)

Return type:

Dict[str, Dict[str, Any]]
