Runtime API¶

This page documents the main runtime entry points and orchestration classes.

Use Modes and Mode Inputs for the user-facing workflow. Use this page when you want the Python modules behind that behavior.

Overview¶

Module

Role

ctrlp.cli

command-line argument parsing and entry point wiring

ctrlp.runner

mode dispatch and runtime execution shell

ctrlp.manager.config

control-file parsing, validation, path resolution, and parameter merge

ctrlp.manager.task_policy

mode policy table for env counts, monitor defaults, and compatibility

ctrlp.agent

algorithm lifecycle, loading, training, evaluation, and demo flow

ctrlp.manager

configuration resolution, sweeps, env setup, and mode orchestration

class ctrlp.agent.Agent(train_params, manager)¶

Bases: object

Agent class for training, evaluating, and demonstrating RL policies.

Variables:

  • train_params – Parameters for training (RL or IL depending on task).

  • manager – Manager object containing configuration and paths.

  • config – Configuration dictionary.

  • paths – Paths dictionary for storing results.

  • names – Names dictionary for environment, task, experiment, and agent.

  • baseline – Boolean indicating if a baseline policy is used.

  • envs – Environments managed by the EnvManager.

  • device – Device to run the policy on (CPU or GPU).

  • use_masking – Boolean indicating if action masking is used.

  • printer – Printer object for displaying run information.

Parameters:

  • train_params (Dict[str, Any])

  • manager (Any)

load(path=None, training=False, disable_monitor=False)¶
