MiniGrid¶

Overview¶

MiniGrid is a discrete grid-filling environment designed for fast RL experiments on coverage and movement efficiency.

It is the lightest built-in CTRL-P environment and the best default choice when you want a project-relevant smoke test without the heavier thermal solver used by ThermoGrid.

What The Agent Controls¶

The agent controls:

  • pointer movement

  • whether filling is active

  • optional spread-width changes through sigma control actions

Typical action sets include movement in cardinal or diagonal directions plus a toggle action.

Default Action Set¶

The shipped default action configuration comes from src/ctrlp/environments/minigrid/defaults/env.yml.

Index

Note

The generic grid action helper also supports diagonal actions such as UL, UR, DL, DR, and STAY, but the shipped MiniGrid default uses the seven-action set above.

Observation Structure¶

MiniGrid observations are built around the current grid state and pointer state.

Depending on configuration, they can include:

  • tensor-like spatial state

  • vector features for pointer state or auxiliary values

By default, MiniGrid exposes three state channels:

Channel

Source

Meaning

phase

derived state

binary fill state of the grid

load

load field

current load footprint; 1 when active, 0.5 when inactive, 0 elsewhere

mask

ProcGen / scenario target

target region that should be filled

For the shipped 10 x 10 task defaults, the internal mesh adds a one-cell border on each side, so the default observation tensor is shaped like 3 x 12 x 12.

Reward Intuition¶

MiniGrid rewards typically encourage:

  • covering new target cells

  • avoiding redundant motion

  • avoiding invalid or wasted actions

  • finishing the task efficiently

The default reward configuration uses these weighted terms:

Term

Default weight

Effect

phase

coverage

1.5

reward new filled area inside the target mask

phase

0.15

phase

1.0

phase

0.15

general

1.5

penalize taking extra steps

score is reported separately from reward and is computed as:

\[ ext{score} = \text{inside-fill-ratio} - \text{outside-fill-ratio} \]

When To Use It¶

MiniGrid is a good fit when you want:

  • a fast built-in environment

  • discrete action-space experiments

  • simple rollout generation

  • algorithm smoke tests that are closer to the real project than CartPole

Tip

Use MiniGrid when you want a project-relevant environment that still trains quickly enough for structural checks and smaller experiments.

Minimal Example¶

CTRL-P already ships working MiniGrid control files under experiments/minigrid/configs/. A minimal control file looks like this:

runtime:
  sweep:
    seed: [0]

  train:
    timesteps: 5000

Typical validation flow:

ctrlp experiments/minigrid/configs/fill/ppo/smoke.yml --mode check
ctrlp experiments/minigrid/configs/fill/ppo/smoke.yml --mode train --envs 1
ctrlp experiments/minigrid/configs/fill/ppo/smoke.yml --mode eval

If you want the exact shipped smoke setup, inspect:

  • experiments/minigrid/parameters/tasks/fill.yml

  • experiments/minigrid/configs/fill/ppo/smoke.yml

  • experiments/minigrid/parameters/algo/ppo/smoke.yml
