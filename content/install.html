Install¶

This page covers the normal setup path for CTRL-P on Linux and WSL. CTRL-P is distributed as a Python package, and the normal workflow is to install it into a virtual environment before using the ctrlp command. Native Windows is not the main path.

Before You Start¶

Note

Before installing CTRL-P, make sure you have:

  • Python 3.10 or newer

  • a terminal or command prompt

  • git, if you plan to work from a local source checkout

If Python is not installed yet, download it from python.org.

To check whether Python is already available on Linux or WSL, run:

python3 --version

If that prints a Python version of 3.10 or newer, you are ready to continue.

Tip

If you do not already have a preferred editor, VS Code is a good default IDE for working with CTRL-P.

If you are on Windows, it is recommended to run the project through WSL instead of plain Windows Python. Microsoft explains how to set it up here: Windows Subsystem for Linux Documentation.

If you have access to GitHub Copilot or ChatGPT with Codex-style coding assistance, using one of those tools can speed up reading, editing, and navigating the codebase.

Standard Setup¶

1. Create a virtual environment¶

Linux / WSL¶

python3 -m venv .venv-wsl
source .venv-wsl/bin/activate

Tip

Use a project-local venv. It makes the docs, CLI, and editor configuration easier to reason about.

2. Install CTRL-P as a package¶

If you are installing a released build or using an internal package index:

pip install ctrlp

If you are working from a local source checkout for development:

pip install -e .

Then install the runtime stack used by the docs and examples:

pip install -r tools/installation/requirements/requirements.wsl.txt

3. Verify the installation¶

ctrlp experiments/cartpole-v1/configs/default/ppo/default.yml --mode resolve
ctrlp experiments/cartpole-v1/configs/default/ppo/default.yml --mode check

Expected outcome:

  • resolve prints merged parameters and paths

  • check builds one env and one model and reports success

Warning

If check fails, do not continue to train. Fix the environment/model setup first.

Note

The CartPole example used in this install page is a local bootstrap example. CTRL-P will create the required files under experiments/cartpole-v1/ the first time you run it, and .gitignore keeps that tree out of version control. Use it for installation checks and quick smoke tests; use dawn, minigrid, or thermogrid when you want the maintained in-repo experiment families.

Optional GPU Setup¶

You do not need this for a normal first installation.

Install a CUDA-enabled PyTorch build if you want GPU training on Linux or WSL.

CUDA 12.1¶

pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121

CPU-only¶

pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu

Verify:

python3 -c "import torch; print(torch.cuda.is_available(), torch.version.cuda)"

Optional JAX Setup¶

You do not need this for a normal first installation.

JAX support is available through the SBX backend on Linux or WSL.

CPU-only¶

pip install sbx-rl jax[cpu]

GPU¶

pip install sbx-rl jax[cuda] -f https://storage.googleapis.com/jax-releases/jax_cuda_releases.html

Use it with:

ctrlp experiments/cartpole-v1/configs/default/ppo/default.yml --mode train --jax

Warning

JAX support should be treated as a second backend, not the first thing to debug. Validate the same config on the standard backend first.

Fastest First Run¶

If you only want to confirm that the installation works:

ctrlp experiments/cartpole-v1/configs/default/ppo/default.yml --mode resolve
ctrlp experiments/cartpole-v1/configs/default/ppo/default.yml --mode check
ctrlp experiments/cartpole-v1/configs/default/ppo/default.yml --mode train --envs 1
ctrlp experiments/cartpole-v1/configs/default/ppo/default.yml --mode eval

This flow bootstraps the CartPole files locally and leaves them ignored by git. It is a good way to verify the installation without touching the maintained experiment trees.

Troubleshooting¶

python3 not found¶

Install Python 3 and make sure it is on your PATH.

Virtual environment creation fails¶

On Debian/Ubuntu:

sudo apt-get install python3-venv

check fails even though installation succeeded¶

Common causes:

  • missing optional dependencies

  • wrong Python interpreter selected in the editor

  • incompatible GPU/JAX installation

  • stale environment files from another branch or venv

Tip

When debugging installation issues, use:

  1. resolve

  2. check

  3. only then train

After Installation¶

Once the environment is working, the normal next steps are:

  1. Experiments for the first runtime path.

  2. Command Guide for fast task lookup.

  3. Environment Guide if you are selecting a task family.

  4. Project Layout if you want to inspect generated files.
